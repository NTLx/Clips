# Wikilinks 短链接格式修复设计文档

## 问题背景

Clips 知识库中的 Wiki 文件使用相对路径链接格式 `[[../00-raw/web-clips/{分类}/{文件名}]]`，该格式在 Obsidian 和 Quartz 网站中都无法正确解析。

## 目标

将所有相对路径链接转换为短链接格式 `[[{文件名}]]`，确保 Obsidian 和 Quartz 都能正确解析。

## 技术分析

### Quartz 配置验证

- 配置文件 `quartz.config.ts` 已设置 `markdownLinkResolution: "shortest"`
- 该模式下，如果文件名唯一，使用纯文件名匹配；如果不唯一，使用完整路径

### 文件名唯一性

- 经验证，所有 Markdown 文件名均唯一（包括 Skills 相关文件已区分）
- 短链接格式可安全使用

### 现有正确格式验证

- `[[Taste]]`、`[[Judgment]]` 等短链接已在多个文件中使用
- Obsidian 和 Quartz 都正确解析

## 转换规则

```
原始格式: [[../00-raw/web-clips/{分类}/{文件名}]]
         [[../00-raw/web-clips/{分类}/{文件名}|别名]]
         [[../index|知识库索引]]

转换后格式: [[{文件名}]]
           [[{文件名}|别名]]
           [[index|知识库索引]]

规则:
1. 提取路径中最后一级文件名
2. 保留别名部分（如有）
3. 不保留 .md 后缀（遵循现有惯例）
```

## 影响范围

| 文件类型 | 文件数 | 链接数 |
|---------|-------|-------|
| Entities | ~60 | ~120 |
| Topics | ~15 | ~60 |
| Comparisons | ~5 | ~20 |
| README.md | 1 | ~5 |
| **总计** | **76** | **208** |

## 实施步骤

1. **扫描验证**: 确认所有相对路径链接位置
2. **批量替换**: 使用脚本转换所有相对路径链接
3. **结果验证**: 确认无残留相对路径链接
4. **链接有效性检查**: 确认所有短链接指向的文件存在
5. **Git 提交**: 提交修复后的文件

## 风险与缓解

| 风险 | 缓解措施 |
|-----|---------|
| 替换遗漏边缘情况 | 正则表达式覆盖多种格式，包括带别名的情况 |
| 特殊字符处理 | 保留原始文件名中的特殊字符（中文、括号等） |
| index.md 链接 | 特殊处理 `[[../index]]` → `[[index]]` |

## 验收标准

- [ ] 所有 `[[../` 格式链接已转换为短链接
- [ ] `grep -r "\[\[\.\./"` 返回 0 结果
- [ ] 所有短链接指向的文件物理存在
- [ ] Quartz 本地构建成功（可选验证）
- [ ] Git 提交完成

## 后续维护
