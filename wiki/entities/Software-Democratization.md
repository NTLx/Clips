# Software Democratization（软件民主化）

> [!definition] 定义
> **Software Democratization** 是 Boris Cherny 借印刷术历史类比提出的趋势预测：15 世纪前欧洲识字率约 10%（识字的 scribes 为不识字统治阶级服务），印刷术发明后 50 年出版量超过去一千年总和，500 年后全球识字率 70%。Boris 认为 AI coding agents 将以远超印刷术的速度复制这一过程——编程从专业技能变成"像发短信一样"的通用能力。

## 关键数据点

- **印刷术类比**：识字率 10% → 70%，书籍成本降 100x，跨度 500 年
- **软件民主化的加速因素**：不需要等教育体系重建、不需要农业社会转型——互联网 + AI 使传播近乎即时
- **领域专家的崛起**：会计写会计软件、医生写医疗软件、教师写教育软件——编程是简单部分，领域知识是难的部分
- **专业化仍然存在**：就像有了印刷术后专业作家仍然存在，"professional developer"不会消失，但不再是唯一能构建软件的人

## 前提与局限性

- AI coding agent 的输出质量目前在边缘场景仍不可靠——需要 human oversight
- 复杂系统（分布式、安全关键）仍然需要专业工程知识
- "人人能编程"不等于"人人能设计好软件"——Taste 和 Judgment 仍然是稀缺能力
